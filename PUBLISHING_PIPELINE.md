# NexusBlue App Publishing Pipeline
## Capacitor + GitHub Actions — Multi-App Strategy

This document describes the recommended publishing pipeline for all future NexusBlue apps.
Set it up once on this repo as the template. Every future app reuses the same approach.

---

## Why This Stack

| Decision | Reason |
|----------|--------|
| Keep HTML/CSS/JS (PWA) | Apps already work, no rewrite needed |
| Capacitor (not Cordova) | Modern, actively maintained, great Web Speech API support |
| GitHub Actions | Free macOS build servers — no Mac hardware needed |
| One developer account per store | Covers unlimited apps under NexusBlue brand |

## Architecture Overview

```
nexusblue-[app-name]/          ← Your PWA repo (HTML/CSS/JS)
├── index.html
├── css/
├── js/
├── capacitor.config.json       ← Added once per app
├── android/                    ← Generated by Capacitor (git-ignored)
├── ios/                        ← Generated by Capacitor (git-ignored)
└── .github/
    └── workflows/
        ├── android.yml         ← Auto-builds + uploads to Google Play
        └── ios.yml             ← Auto-builds + uploads to App Store
```

## Setup Instructions (Do Once Per App)

### Step 1: Install Capacitor

```bash
npm init -y
npm install @capacitor/core @capacitor/cli
npm install @capacitor/android @capacitor/ios
npx cap init "[App Name]" "com.nexusblue.[appname]" --web-dir "."
```

### Step 2: Create capacitor.config.json

```json
{
  "appId": "com.nexusblue.juniorjarviscareer",
  "appName": "Junior Jarvis Career",
  "webDir": ".",
  "server": {
    "androidScheme": "https"
  },
  "plugins": {
    "SplashScreen": {
      "launchShowDuration": 2000,
      "backgroundColor": "#000000"
    }
  }
}
```

### Step 3: Add Platforms

```bash
npx cap add android
npx cap add ios
npx cap sync
```

### Step 4: Add to .gitignore

```
node_modules/
android/
ios/
```

## GitHub Actions — Android Pipeline

Create `.github/workflows/android.yml`:

```yaml
name: Build & Deploy Android

on:
  push:
    branches: [main]
    paths:
      - '**.js'
      - '**.html'
      - '**.css'
      - 'capacitor.config.json'

jobs:
  build-android:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm ci

      - name: Sync Capacitor
        run: npx cap sync android

      - name: Setup Java
        uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: Build Release APK
        run: |
          cd android
          ./gradlew bundleRelease

      - name: Sign AAB
        uses: r0adkll/sign-android-release@v1
        with:
          releaseDirectory: android/app/build/outputs/bundle/release
          signingKeyBase64: ${{ secrets.ANDROID_SIGNING_KEY }}
          alias: ${{ secrets.ANDROID_KEY_ALIAS }}
          keyStorePassword: ${{ secrets.ANDROID_KEY_STORE_PASSWORD }}
          keyPassword: ${{ secrets.ANDROID_KEY_PASSWORD }}

      - name: Upload to Google Play
        uses: r0adkll/upload-google-play@v1
        with:
          serviceAccountJsonPlainText: ${{ secrets.GOOGLE_PLAY_SERVICE_ACCOUNT }}
          packageName: com.nexusblue.juniorjarviscareer
          releaseFiles: android/app/build/outputs/bundle/release/*.aab
          track: internal
```

## GitHub Actions — iOS Pipeline

Create `.github/workflows/ios.yml`:

```yaml
name: Build & Deploy iOS

on:
  push:
    branches: [main]
    paths:
      - '**.js'
      - '**.html'
      - '**.css'
      - 'capacitor.config.json'

jobs:
  build-ios:
    runs-on: macos-latest        # ← GitHub's free macOS runner
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm ci

      - name: Sync Capacitor
        run: npx cap sync ios

      - name: Install Apple Certificates
        uses: apple-actions/import-codesign-certs@v2
        with:
          p12-file-base64: ${{ secrets.IOS_P12_CERTIFICATE }}
          p12-password: ${{ secrets.IOS_P12_PASSWORD }}

      - name: Install Provisioning Profile
        uses: apple-actions/download-provisioning-profiles@v1
        with:
          bundle-id: com.nexusblue.juniorjarviscareer
          issuer-id: ${{ secrets.APPSTORE_ISSUER_ID }}
          api-key-id: ${{ secrets.APPSTORE_KEY_ID }}
          api-private-key: ${{ secrets.APPSTORE_PRIVATE_KEY }}

      - name: Build & Archive
        run: |
          cd ios/App
          xcodebuild archive \
            -scheme App \
            -archivePath $RUNNER_TEMP/App.xcarchive \
            -configuration Release \
            CODE_SIGNING_REQUIRED=YES

      - name: Export IPA
        run: |
          xcodebuild -exportArchive \
            -archivePath $RUNNER_TEMP/App.xcarchive \
            -exportPath $RUNNER_TEMP/App.ipa \
            -exportOptionsPlist ios/ExportOptions.plist

      - name: Upload to App Store Connect
        uses: apple-actions/upload-testflight-build@v1
        with:
          app-path: $RUNNER_TEMP/App.ipa/App.ipa
          issuer-id: ${{ secrets.APPSTORE_ISSUER_ID }}
          api-key-id: ${{ secrets.APPSTORE_KEY_ID }}
          api-private-key: ${{ secrets.APPSTORE_PRIVATE_KEY }}
```

## GitHub Secrets Required

Set these in each repo under **Settings → Secrets and variables → Actions**:

### Android Secrets
| Secret Name | How to Get |
|-------------|------------|
| `ANDROID_SIGNING_KEY` | Generate a keystore: `keytool -genkey -v -keystore nexusblue.keystore -alias nexusblue -keyalg RSA -keysize 2048 -validity 10000`, then base64 encode it |
| `ANDROID_KEY_ALIAS` | The alias you used above (`nexusblue`) |
| `ANDROID_KEY_STORE_PASSWORD` | Password you set on the keystore |
| `ANDROID_KEY_PASSWORD` | Key password (can be same as above) |
| `GOOGLE_PLAY_SERVICE_ACCOUNT` | JSON key from Google Play Console → Setup → API access |

### iOS Secrets
| Secret Name | How to Get |
|-------------|------------|
| `IOS_P12_CERTIFICATE` | Export from Apple Developer portal, base64 encode |
| `IOS_P12_PASSWORD` | Password set when exporting the certificate |
| `APPSTORE_ISSUER_ID` | App Store Connect → Users → Keys → Issuer ID |
| `APPSTORE_KEY_ID` | App Store Connect → Users → Keys → Key ID |
| `APPSTORE_PRIVATE_KEY` | Download from App Store Connect (only available once) |

## Reusing This Pipeline for New Apps

When building the next NexusBlue app:

1. Clone this repo as a starting point
2. Change `appId` in `capacitor.config.json` (`com.nexusblue.[newappname]`)
3. Update `packageName` in `android.yml` and `bundle-id` in `ios.yml`
4. Add the same GitHub secrets to the new repo (Android keystore is reusable across apps)
5. Push to `main` — both stores get the build automatically

**The Android keystore (`nexusblue.keystore`) should be saved securely — it is used to sign ALL NexusBlue Android apps. Never lose it.**

## Web Speech API Notes for App Stores

The voice feature works differently in Capacitor than in a browser:

- **Android**: Works natively via WebView. No changes needed.
- **iOS**: Requires the `NSMicrophoneUsageDescription` and `NSSpeechRecognitionUsageDescription` keys in `Info.plist` even for TTS-only apps. Add to `ios/App/App/Info.plist`:

```xml
<key>NSMicrophoneUsageDescription</key>
<string>Used for voice interaction with Junior Jarvis</string>
```

- The tap-overlay in the app already handles the required user gesture for iOS speech unlock.
